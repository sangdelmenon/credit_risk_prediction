- Confusion Matrix: True Positives, False Positives, False Negatives, True Negatives

Step 9: MODEL INTERPRETATION
- Feature importance: Which features matter most?
- SHAP values: How does each feature impact individual predictions?
- Partial dependence plots: How does changing one feature affect predictions?

Step 10: DEPLOYMENT PREPARATION

DECISION 2: Why SMOTE for Class Imbalance?

CHOICE: Use SMOTE (Synthetic Minority Over-sampling Technique)

PROBLEM: Imbalanced dataset (20% default, 80% non-default)
- Model learns to predict majority class (non-default) always
- Achieves 80% accuracy but misses all defaults!
- Business impact: Approve bad loans, lose money

SMOTE SOLUTION:
- Creates synthetic minority class samples
- Balances training data to 50-50
- Model learns patterns in both classes

HOW SMOTE WORKS:
1. Find k nearest neighbors for each minority sample
2. Create synthetic sample between original and neighbor
3. Repeat until classes balanced

ALTERNATIVES CONSIDERED:
- Undersampling: Remove majority class samples
  REJECTED: Loses information
- Class weights: Penalize misclassifying minority class
  USED IN ADDITION: Set class_weight='balanced'
- Ensemble methods: Easy Ensemble, Balanced Random Forest
  FUTURE ENHANCEMENT

CODE EXAMPLE:
```python
from imblearn.over_sampling import SMOTE

smote = SMOTE(random_state=42)
X_train_balanced, y_train_balanced = smote.fit_resample(X_train, y_train)

# Before SMOTE: 80,000 non-default, 20,000 default
# After SMOTE: 80,000 non-default, 80,000 default (synthetic)
```

CRITICAL: Only apply SMOTE to training data, NOT validation/test
- Why: Validation/test should reflect real-world distribution

REASONS:
- Raw features have limited predictive power

5. Behavioral Features:

DECISION 4: Why AUC-ROC Instead of Accuracy?

CHOICE: Use AUC-ROC as primary metric, not accuracy

PROBLEM WITH ACCURACY:
- Imbalanced dataset: 80% non-default, 20% default
- Model that predicts "non-default" always: 80% accuracy!
- But catches 0% of defaults (useless for business)

AUC-ROC ADVANTAGES:
- Robust to class imbalance
- Measures model's ability to distinguish classes
- Range: 0.5 (random) to 1.0 (perfect)
- Interpretable: Probability that model ranks random default higher than random non-default

FORMULA:
AUC-ROC = Area Under Receiver Operating Characteristic Curve
- X-axis: False Positive Rate (Type I error)
- Y-axis: True Positive Rate (Recall)

BUSINESS INTERPRETATION:
- AUC 0.86 means: "If we show the model one default and one non-default, 
  it will correctly identify the default 86% of the time"

OTHER METRICS TRACKED:
- Precision: Of predicted defaults, how many are correct?
  (Cost of false alarms)
- Recall: Of actual defaults, how many did we catch?
  (Cost of missed defaults)
- F1-Score: Balance between precision and recall

ALTERNATIVE METRICS:
- Accuracy: Misleading for imbalanced data
- Precision-Recall AUC: Better for extreme imbalance (>95%)
- Matthews Correlation Coefficient: Good for imbalanced data

DECISION 5: Why Stratified K-Fold Cross-Validation?

CHOICE: Use stratified k-fold (k=5), not simple train-test split

SIMPLE SPLIT PROBLEMS:
- Results depend on random split
- Might get lucky/unlucky with split
- Doesn't use all data for training

CROSS-VALIDATION BENEFITS:
- Use all data for both training and validation
- Average over 5 folds reduces variance
- More robust performance estimate

STRATIFIED K-FOLD:
- Regular k-fold: Splits randomly, might get 5% defaults in one fold, 30% in another
- Stratified: Ensures each fold has same default rate (~20%)
- Critical for imbalanced data

HOW IT WORKS:
1. Split data into 5 folds, each with 20% default rate
2. Train on folds 1-4, validate on fold 5
3. Train on folds 1-3,5, validate on fold 4
4. Repeat for all folds
5. Average performance across 5 folds

CODE EXAMPLE:
```python
from sklearn.model_selection import StratifiedKFold

skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

scores = []
for train_idx, val_idx in skf.split(X, y):
    X_train_fold, X_val_fold = X[train_idx], X[val_idx]
    y_train_fold, y_val_fold = y[train_idx], y[val_idx]
    
    model.fit(X_train_fold, y_train_fold)
    score = roc_auc_score(y_val_fold, model.predict_proba(X_val_fold)[:, 1])
    scores.append(score)

mean_auc = np.mean(scores)
std_auc = np.std(scores)
print(f"AUC: {mean_auc:.3f} ± {std_auc:.3f}")
```

DECISION 6: Why Grid Search for Hyperparameter Tuning?

CHOICE: Use GridSearchCV to find optimal hyperparameters

PROBLEM: Default hyperparameters rarely optimal
- XGBoost has 30+ hyperparameters
- Manual tuning tedious and suboptimal

GRID SEARCH APPROACH:
- Define parameter grid
- Try all combinations
- Use cross-validation to evaluate each
- Select best combination

PARAMETERS TUNED:

XGBoost:
- max_depth: [3, 5, 7] (tree depth)
- learning_rate: [0.01, 0.1, 0.3] (step size)
- n_estimators: [100, 200, 300] (number of trees)
- min_child_weight: [1, 3, 5] (minimum samples in leaf)
Total combinations: 3 × 3 × 3 × 3 = 81

Random Forest:
- n_estimators: [100, 200, 300]
- max_depth: [10, 20, 30, None]
- min_samples_split: [2, 5, 10]
Total combinations: 3 × 4 × 3 = 36

COMPUTATIONAL COST:
- 81 combinations × 5 folds = 405 model trainings
- Time: 5-15 minutes on modern CPU

ALTERNATIVES:
- Random Search: Sample random combinations (faster)
- Bayesian Optimization: Smart search (more efficient)
- Optuna: Modern hyperparameter optimization

CHOSEN GRID SEARCH BECAUSE:
- Comprehensive (tries all combinations)
- Simple to implement
- Reproducible results

FUTURE ENHANCEMENT: Use Optuna for faster tuning

DECISION 7: Why SHAP for Model Interpretation?

CHOICE: Use SHAP (SHapley Additive exPlanations) values

BUSINESS NEED: "Why was this loan rejected?"
- Regulators require explainability
- Applicants deserve explanations
- Model validation requires understanding

SHAP ADVANTAGES:
- Theoretically grounded (game theory)
- Works for any model (tree-based, neural nets, etc.)
- Local explanations (individual predictions)
- Global explanations (overall feature importance)

HOW SHAP WORKS:
- Calculates contribution of each feature to prediction
- For each prediction, shows how each feature pushed it higher/lower
- Based on Shapley values from cooperative game theory

EXAMPLE OUTPUT:
Prediction: Default probability = 0.73 (HIGH RISK)

SHAP values:
  Debt-to-Income Ratio (0.45): +0.25 (increases risk)
  Credit Score (680): +0.10 (moderate risk)
  Employment Length (3 yrs): -0.05 (decreases risk)
  Age (28): +0.08 (young = risky)
  Loan Amount ($25k): +0.12 (increases risk)
  Base value: 0.28
  Final prediction: 0.28 + 0.25 + 0.10 - 0.05 + 0.08 + 0.12 = 0.78 ≈ 0.73

INTERPRETATIONS:
- "Main reason for rejection: Debt-to-Income ratio of 45% is too high"
- "Credit score of 680 is moderate, contributes to risk"
- "3 years employment helps, but not enough to offset other factors"

ALTERNATIVES:
- LIME: Local Interpretable Model-agnostic Explanations
- Feature Importance: Shows overall importance, not individual predictions
- Partial Dependence Plots: Shows how changing one feature affects predictions

CHOSEN SHAP BECAUSE:
- Industry standard
- Mathematically sound
- Works for all models
- Both local and global explanations

DECISION 8: Why XGBoost and LightGBM?

CHOICE: Include gradient boosting models (XGBoost, LightGBM)

GRADIENT BOOSTING ADVANTAGES:
- State-of-the-art performance
- Handles non-linear relationships
- Built-in regularization
- Feature importance
- Robust to outliers

XGBOOST:
- "Extreme Gradient Boosting"
- Won 17 of 29 Kaggle competitions in 2015
- Regularization prevents overfitting
- Parallel processing (fast)
- Handles missing values

LIGHTGBM:
- Developed by Microsoft
- Faster than XGBoost (10x on large datasets)
- Lower memory usage
- Leaf-wise tree growth (more accurate)
- Built-in categorical feature support

HOW GRADIENT BOOSTING WORKS:
1. Start with weak model (e.g., shallow tree)
2. Calculate errors on training data
3. Train new model to predict errors
4. Add to ensemble
5. Repeat 100-1000 times
6. Final prediction = sum of all models

DECISION 9: Why 70-15-15 Split?

CHOICE: 70% training, 15% validation, 15% test

RATIONALE:
- Training (70%): Learn patterns
- Validation (15%): Tune hyperparameters, model selection
- Test (15%): Final unbiased evaluation

WHY NOT 80-20 (no validation)?
- Would tune hyperparameters on test set
- Leads to overfitting to test set
- Test metrics would be optimistic

WHY NOT 60-20-20 (more validation/test)?
- Less training data
- Could work for very large datasets (millions)
- 70-15-15 is good for 100k records

ALTERNATIVE: Cross-validation only (no validation set)
- Use cross-validation for everything
- Could work, but slower
- Validation set faster for quick experiments

DECISION 10: Why Synthetic Data Generation?

CHOICE: Generate synthetic loan data instead of using real data

REASONS:
- Privacy: Real loan data is sensitive (PII)
- Availability: Real datasets hard to find, often proprietary
- Control: Can create specific scenarios
- Reproducibility: Anyone can run the project
- Portfolio purposes: Demonstrates data generation skills

SYNTHETIC DATA CHARACTERISTICS:
- Realistic distributions (income, age, credit score)
- Correlations match real data (income ↔ loan amount)
- Imbalanced classes (20% default, like real data)
- Includes missing values (realistic)
- 100,000+ records (enough for ML)

GENERATION APPROACH:
```python
import numpy as np

# Income: Log-normal distribution
income = np.random.lognormal(mean=10.8, sigma=0.5, size=100000)
# Mean ~$50k, range $20k-$150k

# Age: Normal distribution
age = np.random.normal(loc=38, scale=12, size=100000).clip(18, 80)

# Credit Score: Normal distribution
credit_score = np.random.normal(loc=680, scale=80, size=100000).clip(300, 850)

# Loan Amount: Correlated with income
loan_amount = income * np.random.uniform(0.3, 0.8, size=100000)

# Default probability (logistic function)
debt_to_income = loan_amount / income
prob_default = 1 / (1 + np.exp(-5 * (debt_to_income - 0.4)))
default = np.random.binomial(1, prob_default)
```

CHOSEN SYNTHETIC BECAUSE:
- Full control over features

```python
def predict_loan_application(applicant_data):
    """
    Predict default probability for new loan application
    
    Args:
        applicant_data: dict with keys matching training features
        
    Returns:
        dict with prediction, probability, risk_class
    """
    # Load saved model
    with open('./models/best_model.pkl', 'rb') as f:
        model = pickle.load(f)
    
    # Convert to DataFrame
    df = pd.DataFrame([applicant_data])
    
    # Feature engineering (same as training)
    df = feature_engineer.transform(df)
    
    # Predict
    prob_default = model.predict_proba(df)[0, 1]
    prediction = 1 if prob_default > 0.5 else 0
    
    # Risk classification
    if prob_default < 0.3:
        risk_class = "LOW RISK"
    elif prob_default < 0.6:
        risk_class = "MEDIUM RISK"
    else:
        risk_class = "HIGH RISK"
    
    return {
        'prediction': 'DEFAULT' if prediction == 1 else 'NO DEFAULT',
        'default_probability': prob_default,
        'risk_class': risk_class
    }
```

"""
Credit Risk Prediction - Data Loader Module
Generates synthetic loan data and performs EDA
"""

import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns
from datetime import datetime, timedelta
from typing import Tuple


class LoanDataGenerator:
    """Generate realistic synthetic loan application data"""
    
    def __init__(self, n_samples: int = 100000, random_state: int = 42):
        """
        Initialize data generator
        
        Args:
            n_samples: Number of loan applications to generate
            random_state: Random seed for reproducibility
        """
        self.n_samples = n_samples
        np.random.seed(random_state)
        self.data = None
        
    def generate_data(self) -> pd.DataFrame:
        """
        Generate complete loan dataset with realistic distributions
        
        Returns:
            DataFrame with loan applications
        """
        print(f"Generating synthetic loan dataset with {self.n_samples:,} records...")
        
        # Demographics
        age = np.random.normal(loc=38, scale=12, size=self.n_samples).clip(18, 80)
        
        # Income (log-normal distribution, realistic for income)
        income = np.random.lognormal(mean=10.8, sigma=0.5, size=self.n_samples)
        income = income.clip(20000, 200000)  # $20k - $200k range
        
        # Employment length (correlated with age)
        employment_length = (age - 18) * np.random.uniform(0.3, 0.7, size=self.n_samples)
        employment_length = employment_length.clip(0, 40)
        
        # Credit score (normal distribution)
        credit_score = np.random.normal(loc=680, scale=80, size=self.n_samples)
        credit_score = credit_score.clip(300, 850)
        
        # Loan amount (correlated with income)
        loan_amount = income * np.random.uniform(0.3, 0.8, size=self.n_samples)
        
        # Interest rate (higher for lower credit scores)
        base_rate = 0.06  # 6% base
        credit_penalty = (750 - credit_score) / 1000  # Higher penalty for lower scores
        interest_rate = base_rate + credit_penalty.clip(0, 0.15)
        
        # Loan term (months)
        loan_term = np.random.choice([12, 24, 36, 48, 60], size=self.n_samples, 
                                     p=[0.1, 0.2, 0.4, 0.2, 0.1])
        
        # Monthly payment
        r = interest_rate / 12  # Monthly rate
        n = loan_term
        monthly_payment = loan_amount * (r * (1 + r)**n) / ((1 + r)**n - 1)
        
        # Credit utilization
        credit_limit = income * np.random.uniform(0.5, 2.0, size=self.n_samples)
        credit_used = credit_limit * np.random.beta(2, 5, size=self.n_samples)
        
        # Delinquencies (Poisson distribution)
        num_delinquencies = np.random.poisson(lam=0.5, size=self.n_samples)
        
        # Inquiries (related to credit seeking behavior)
        num_inquiries = np.random.poisson(lam=1.5, size=self.n_samples)
        
        # Existing debt (correlated with income)
        existing_debt = income * np.random.uniform(0.1, 0.6, size=self.n_samples)
        
        # Calculate default probability (logistic function of risk factors)
        debt_to_income = (loan_amount + existing_debt) / income
        payment_to_income = monthly_payment / (income / 12)
        credit_util = credit_used / credit_limit
        
        # Risk score (higher = more risky)
        risk_score = (
            5 * (debt_to_income - 0.4) +
            3 * (payment_to_income - 0.2) +
            2 * (credit_util - 0.3) +
            -0.005 * (credit_score - 680) +
            0.5 * num_delinquencies +
            0.2 * num_inquiries +
            -0.05 * employment_length
        )
        
        # Convert to probability using logistic function
        prob_default = 1 / (1 + np.exp(-risk_score))
        
        # Generate actual defaults
        default = np.random.binomial(1, prob_default)
        
        # Add some missing values (realistic)
        def add_missing(arr, missing_rate=0.02):
            mask = np.random.random(len(arr)) < missing_rate
            arr = arr.copy()
            arr[mask] = np.nan
            return arr
        
        # Create DataFrame
        self.data = pd.DataFrame({
            'age': age,
            'income': add_missing(income, 0.01),
            'employment_length': add_missing(employment_length, 0.03),
            'credit_score': add_missing(credit_score, 0.02),
            'loan_amount': loan_amount,
            'interest_rate': interest_rate,
            'loan_term': loan_term,
            'monthly_payment': monthly_payment,
            'credit_limit': credit_limit,
            'credit_used': credit_used,
            'num_delinquencies': num_delinquencies,
            'num_inquiries': num_inquiries,
            'existing_debt': existing_debt,
            'debt_to_income': debt_to_income,
            'payment_to_income': payment_to_income,
            'default': default
        })
        
        default_rate = self.data['default'].mean() * 100
        print(f"✓ Generated {len(self.data):,} loan applications")
        print(f"  Default rate: {default_rate:.1f}%")
        print(f"  Features: {len(self.data.columns)}")
        
        return self.data

    def split_data(self, test_size: float = 0.15, val_size: float = 0.15) -> Tuple:
        """
        Split data into train, validation, and test sets
        
        Args:
            test_size: Proportion for test set
            val_size: Proportion for validation set
            
        Returns:
            Tuple of (X_train, X_val, X_test, y_train, y_val, y_test)
        """
        from sklearn.model_selection import train_test_split
        
        if self.data is None:
            raise ValueError("Generate data first using generate_data()")
        
        # Separate features and target
        X = self.data.drop('default', axis=1)
        y = self.data['default']
        
        # First split: separate test set
        X_temp, X_test, y_temp, y_test = train_test_split(
            X, y, test_size=test_size, random_state=42, stratify=y
        )
        
        # Second split: separate train and validation
        val_size_adjusted = val_size / (1 - test_size)  # Adjust for remaining data
        X_train, X_val, y_train, y_val = train_test_split(
            X_temp, y_temp, test_size=val_size_adjusted, random_state=42, stratify=y_temp
        )
        
        print(f"\nData Split:")
        print(f"  Training: {len(X_train):,} ({len(X_train)/len(X)*100:.1f}%)")
        print(f"  Validation: {len(X_val):,} ({len(X_val)/len(X)*100:.1f}%)")
        print(f"  Test: {len(X_test):,} ({len(X_test)/len(X)*100:.1f}%)")
        print(f"\n  Training default rate: {y_train.mean()*100:.1f}%")
        print(f"  Validation default rate: {y_val.mean()*100:.1f}%")
        print(f"  Test default rate: {y_test.mean()*100:.1f}%")
        
        return X_train, X_val, X_test, y_train, y_val, y_test
